### Алгоритм решения
~~Запрашиваем у пользователя количество элементов последовательности с помощью функции readSize.
если введено некорректное число или отрицательное, повторяем ввод рекурсивно.

Создаём ленивую последовательность seq { for _ in 1 .. size -> readElement() }.
Здесь ленивость проявляется в том, что элементы читаются и обрабатываются по мере их запроса, а не создаётся весь список сразу.

С помощью Seq.fold проходим по этой ленивой последовательности один раз, одновременно формируя:
исходную последовательность numbersOutput
последовательность последних цифр digitsOutput (через функцию convertIntString)
Этот шаг тоже использует ленивость, потому что fold обрабатывает элементы по мере их появления, не требуя хранения всех элементов в памяти.

Выводим на экран результаты:
исходную последовательность
последовательность последних цифр каждого числа~~

1.readSize() - запрашиваем размер последовательности, проверяем, что введено целое число >=0.

2. Создаём ленивую последовательность чисел: seq { for _ in 1 .. size -> readElement() }.
Здесь ленивость: числа ещё не запрашиваются, readElement() вызовется только при проходе по seq.

3.Для каждого элемента последовательности:
  запрашиваем число (readElement)
  вычисляем его последнюю цифру через convertIntString
  выводим на экран сам элемент
  выводим на экран последнюю цифру
4. при map нужен Seq.toArray  чтобы все элементы реально обработались(нужна проходка чтобы ленивые вычисл начали вычисляться)

# Задание 3

## Задача 1

### Текст задачи

Решить с использованием последовательности задачу на обработку файлов/каталогов

Вывести пути ко всем файлам, не являющимся текстовыми (имеют

расширение, отличное от txt), в указанном каталоге и его подкаталогах.

### Алгоритм решения



Пользователь вводит путь к каталогу через функцию askDirectory.

Проверяется существование каталога с помощью Directory.Exists.

Если каталог не существует, программа повторно запрашивает путь.



Создаются безопасные функции safeFiles и safeDirectories, которые возвращают последовательности файлов и подкаталогов.

Эти функции обрабатывают ошибки доступа и возвращают пустую последовательность в случае недоступных элементов.



Определяется рекурсивная функция getFiles, которая возвращает ленивую последовательность (seq<string>) всех файлов в каталоге и его подкаталогах.

Используется генератор seq { ... } с yield!:

\- yield! safeFiles path -добавляет файлы текущего каталога.

\- for subdir in safeDirectories path do yield! getFiles subdir - рекурсивно добавляет файлы всех подкаталогов.

Ленивость здесь заключается в том, что файлы и подкаталоги считываются только при переборе элементов, а не сразу.



Создаётся переменная allFiles, которая хранит последовательность всех файлов, возвращаемую getFiles.

В этот момент файлы ещё не считываются, последовательность только подготовлена.



Применяется Seq.filter для отбора файлов, расширение которых не .txt.

Функция Seq.filter также ленивая: проверка каждого файла выполняется только тогда, когда элемент реально перебирается.



Для вывода используется Seq.iter, который перебирает последовательность и выполняет printfn "%s" для каждого файла

&nbsp;здесь начинается фактическое чтение файлов и подкаталогов, проверка расширений и вывод на экран

Все предыдущие операции (генерация и фильтрация последовательностей) до этого момента оставались ленивыми
