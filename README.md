# Задание 1

## Задача 1

### Текст задачи

Решить задачу из лабораторной работы №2 (Задание 1. List.map Получить список из последних цифр чисел, содержащихся в исходном списке) для

последовательности.

### Алгоритм решения

1 Пользователь вводит количество элементов последовательности.

Функция readSize() считывает строку, преобразует её в целое число и проверяет корректность ввода. При ошибке функция вызывает себя повторно.



2 Пользователь вводит элементы последовательности.

Функция readElement() считывает значение, пытается преобразовать его в число типа float и при ошибке снова вызывает себя.



3  Создаётся массив элементов.

Используется Array.init size (fun \_ -> readElement()).

Функция Array.init вызывает readElement() size раз, поэтому пользователь вводит все элементы сразу. Результат сохраняется в массиве buffer.



4  Создаётся последовательность из массива.

Используется buffer |> Seq.ofArray, где Seq.ofArray преобразует массив в последовательность numbers.



5  Формируется последовательность последних цифр.

Используется numbers |> Seq.map convertintString.

Функция Seq.map создаёт новую последовательность lastDigits, применяя функцию convertintString к каждому элементу.



Здесь возникают ленивые вычисления.

Seq.map не выполняет функцию convertintString сразу, а только создаёт ленивое описание преобразования элементов последовательности.



7  Выводится исходная последовательность.



8  Выводится последовательность последних цифр.



Ленивые вычисления происходят при обходе последовательности lastDigits.

Во время Seq.iter для каждого элемента берётся число из numbers, применяется функция convertintString, и полученная последняя цифра выводится на экран.

# Задание 2

## Задача 1

### Текст задачи

Решить задачу из лабораторной работы №2 (Задание 2. List.fold Найти сумму тех элементов списка, которые оканчиваются на заданную цифру ) для

последовательности

### Алгоритм решения



1\.Пользователь вводит количество элементов последовательности.  Если введено неверное значение, функция рекурсивно вызывает себя до получения корректного числа.



2. Пользователь вводит элементы последовательности. Если введён неверный формат, функция снова вызывает себя.



3.Создаётся массив элементов.

Используется Array.init size (fun \_ -> readElement()).

Функция Array.init создаёт массив размером size и вызывает функцию readElement() для каждого элемента. В этот момент пользователь вводит все числа, и они сохраняются в массиве buffer.



4\. Из массива создаётся последовательность.  создаёт последовательность numbers типа seq<int>.



5  .Пользователь вводит цифру, на которую должны оканчиваться числа.

Функция readDigit() считывает значение, проверяет что оно находится в диапазоне от 0 до 9 и при ошибке повторяет ввод.



6 .Начинается вычисление суммы элементов последовательности.

Вызывается функция sumByDigit numbers digit, внутри которой используется Seq.fold.



7\. Функция Seq.fold проходит по всей последовательности numbers и накапливает сумму.



8\.  Функция endsWithDigit проверяет, оканчивается ли число на заданную цифру.

Число преобразуется к целому типу, берётся его последняя цифра через операцию absInt % 10, и она сравнивается с введённой цифрой.



&nbsp;ленивые вычисления происходят при обходе последовательности функцией Seq.fold.

Последовательность numbers сама по себе не выполняет никаких вычислений заранее.

Когда Seq.fold начинает работу, он по очереди запрашивает элементы последовательности и для каждого элемента выполняет проверку endsWithDigit и обновляет сумму.



После того как Seq.fold обработает все элементы последовательности, полученная сумма сохраняется в переменной result и выводится на экран.

# Задание 3

## Задача 1

### Текст задачи

Решить с использованием последовательности задачу на обработку файлов/каталогов

Вывести пути ко всем файлам, не являющимся текстовыми (имеют

расширение, отличное от txt), в указанном каталоге и его подкаталогах.

### Алгоритм решения



Пользователь вводит путь к каталогу через функцию askDirectory.

Проверяется существование каталога с помощью Directory.Exists.

Если каталог не существует, программа повторно запрашивает путь.



Создаются безопасные функции safeFiles и safeDirectories, которые возвращают последовательности файлов и подкаталогов.

Эти функции обрабатывают ошибки доступа и возвращают пустую последовательность в случае недоступных элементов.



Определяется рекурсивная функция getFiles, которая возвращает ленивую последовательность (seq<string>) всех файлов в каталоге и его подкаталогах.

Используется генератор seq { ... } с yield!:

\- yield! safeFiles path -добавляет файлы текущего каталога.

\- for subdir in safeDirectories path do yield! getFiles subdir - рекурсивно добавляет файлы всех подкаталогов.

Ленивость здесь заключается в том, что файлы и подкаталоги считываются только при переборе элементов, а не сразу.



Создаётся переменная allFiles, которая хранит последовательность всех файлов, возвращаемую getFiles.

В этот момент файлы ещё не считываются, последовательность только подготовлена.



Применяется Seq.filter для отбора файлов, расширение которых не .txt.

Функция Seq.filter также ленивая: проверка каждого файла выполняется только тогда, когда элемент реально перебирается.



Для вывода используется Seq.iter, который перебирает последовательность и выполняет printfn "%s" для каждого файла

&nbsp;здесь начинается фактическое чтение файлов и подкаталогов, проверка расширений и вывод на экран

Все предыдущие операции (генерация и фильтрация последовательностей) до этого момента оставались ленивыми
